The loop

Observe, reason, act — then loop.

The agent has no script and no hardcoded taps. It reads the live screen, decides, and moves — adapting to whatever the app throws at it.

👁

Eyes

describe_screen returns every text element on screen with exact tap coordinates, via on-device OCR.

🧠

Brain

Your LLM reads the profile, weighs it against your taste, and picks the next move — local Gemma or a cloud model.

✋

Hands

tap, swipe, type_text drive the real phone, then it loops back to look again.

Honest by design. A static script can do the hand-movements; only an LLM in the loop can judge a person. So mechanics live in reusable skills, judgment stays with the brain.
Packaged flows

The hand-movements, captured once.

Repeatable mechanics are saved as portable skill files the agent follows — OCR-based steps, not brittle coordinates. The brain just decides which to call.

Bring your own brain

Free and local, or a few cents of cloud.

The agent is just an MCP client. Point it at a local model to spend nothing, or a small cloud model for sharper photo judgement. One line in .env.

Local · free

Ollama + Gemma 3. Runs fully offline. Bigger model = better judgement (16 GB+ recommended).

GOOSE_PROVIDER=ollama

Cloud · cheap

GPT-4o-mini, Gemini Flash or Haiku — strong vision, pennies per run, light on RAM.

GOOSE_MODEL=gpt-4o-mini

Five minutes

Install & run.

Requires macOS 15+ and an iPhone that supports iPhone Mirroring.

1

Clone & configure

Pick your model and behaviour in .env.

$ git clone https://github.com/kairothq/hinge-autoswiper
$ cp .env.example .env # edit model + mode
2

Run the installer

Sets up mirroir, the agent runtime, and your model.

$ ./install.sh
3

Smoke-test on YouTube

Proves the loop with zero risk before anything else.

$ ./run.sh test
4

Go live

Open the app on the mirrored phone and run. Cautious pacing and a session cap are on by default.

$ ./run.sh
Security-first

It only does what you allow.

Giving an AI your phone demands defense in depth. Every layer is fail-closed.

Tool permissions

With no config, only read-only tools exist. Tap/type are hidden from the model until you opt in.

App blocking

Wallet, Banking, Messages, Mail and Photos are blocked — the agent can’t touch them even if enabled.

Kill switch

Close the iPhone Mirroring window and all input stops instantly. No daemons, no root.

{
  "allow": ["tap", "swipe", "type_text", "scroll_to", "launch_app"],
  "deny": [],
  "blockedApps": ["Wallet", "Banking", "Messages", "Photos"]
}
Before you ask

FAQ

Is this against the dating app’s rules?

Yes — automating most dating apps violates their Terms of Service, and accounts can be flagged or banned. The project paces actions like a human and caps each session, which lowers but does not remove the risk. Use a non-critical account and your own judgement.

Does it need my password or a fake API?

No. It works at the screen level through iPhone Mirroring on a phone you’re already logged into. Nothing is stored, and there’s no reverse-engineered API.

Can it run completely free?

Yes, with a local model via Ollama. Photo judgement improves with a larger model or a cheap cloud one; the choice is one line of config.

Does it work with any app?

It operates at the screen level — if you can see it, the agent can read and tap it. No SDK or jailbreak required.

What’s it built on?

mirroir for iPhone control and Goose as an optional local-model runner. Both open source.
